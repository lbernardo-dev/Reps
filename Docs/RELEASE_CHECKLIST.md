# Reps - Internal Release Checklist

## Build

- [ ] Confirm bundle id is `com.romerodev.repsfitness`.
- [ ] Set Apple development team.
- [ ] Build Debug on latest iPhone simulator.
- [ ] Build Release on generic iOS device.
- [ ] Run unit tests.

## Product QA

- [ ] Complete onboarding in English.
- [ ] Complete onboarding in Spanish.
- [ ] Create a custom plan with selected exercises.
- [ ] Activate plan and confirm schedule generation.
- [ ] Start workout from Today.
- [ ] Edit weight/reps and complete sets.
- [ ] Add exercise notes.
- [ ] Finish workout and verify summary.
- [ ] Confirm workout appears in Progress.
- [ ] Schedule workout from Calendar.
- [ ] Add body weight and height manually.
- [ ] Toggle kg/lb and cm/in.

## Device QA

- [ ] Test Apple Health authorization on a real iPhone.
- [ ] Import latest body weight and height from Apple Health.
- [ ] Save latest body weight and height to Apple Health.
- [ ] Enable workout reminders and confirm notification permission.
- [ ] Verify notification delivery.

## Accessibility

- [ ] VoiceOver labels for active workout controls.
- [ ] Dynamic Type does not break key screens.
- [ ] Spanish text does not truncate primary actions.
- [ ] Light and dark mode contrast is acceptable.
- [ ] Tap targets are at least 44pt.

## App Store Preparation

- [ ] Final app icon.
- [ ] Launch screen polish.
- [ ] Privacy policy covering HealthKit data.
- [ ] HealthKit usage copy reviewed.
- [ ] Screenshots in English and Spanish.
